Windows — pakiranje v .exe

📦
Orodje: PyInstaller
Ustvari standalone .exe ki ne potrebuje nameščenega Pythona
📁
Rezultat
dist/Radiola/Radiola.exe + vse odvisnosti v eni mapi
⏱
Čas gradnje
~2–5 minut pri prvi gradnji, ~1 min naslednjič

1. Namesti orodja (enkrat)

# V ukazni vrstici (CMD ali PowerShell)
pip install pyinstaller pillow requests

2. Postavi datoteke

Prepričaj se da so vse datoteke v isti mapi:

Radiola/
├── radiola.py          ← glavni program
├── radiola.spec        ← build konfiguracija
├── build.py            ← avtomatski build skript
├── requirements.txt
└── ICONS/              ← mapa za ikone (lahko prazna)

3. Zaženi avtomatski build

# Opcija A: avtomatski build skript (priporočeno)
python build.py

# Opcija B: direktno PyInstaller
pyinstaller radiola.spec --clean --noconfirm

4. Rezultat

dist/
└── Radiola/
    ├── Radiola.exe     ← glavna izvršilna datoteka
    ├── _internal/      ← Python runtime + odvisnosti
    └── ICONS/          ← mapa za ikone
Distribucija: Celotno mapo dist/Radiola/ zapakiraj v ZIP in jo pošlji naprej. Radiola.exe ne deluje brez _internal/ mape!

5. Namestitev mpv na ciljnem računalniku

Prenesi mpv za Windows z mpv.io/installation → Windows builds
Razpakira in kopiraj mpv.exe v isto mapo kot Radiola.exe
Alternativa: dodaj mpv v sistemski PATH da deluje globalno

Pogoste težave

NapakaRešitev
ModuleNotFoundError: PILpip install pillow
Antivirusni alarmNormalno za PyInstaller exe-je — dodaj izjemo ali podpiši exe
Exe se takoj zapreZaženi iz CMD da vidiš napako: cd dist\Radiola && Radiola.exe
Manjkajoče ikone v oknuPreveri da je _internal/ mapa zraven exe-ja
No player foundKopiraj mpv.exe v mapo Radiola.exe

Opcijsko: ena datoteka (--onefile)

# Ustvari en sam .exe (večji, počasnejši zagon ~5s)
pyinstaller radiola.py --onefile --windowed --name Radiola
Priporočamo mapo namesto --onefile — zagon je takojšen in ni težav z antivirusom.

Linux — binarni program + AppImage

🐧
PyInstaller binary
Deluje na isti distribuciji (Ubuntu 22.04 → samo Ubuntu 22.04+)
📦
AppImage
Ena datoteka, deluje na vseh modernih Linux distribucijah — priporočeno za distribucijo

1. Namesti odvisnosti

# Python paketi
pip install pyinstaller pillow requests --break-system-packages

# Sistemske odvisnosti (Ubuntu/Debian)
sudo apt install python3-tk libpython3-dev

2. Build

# Avtomatski build skript
python3 build.py

# Ali direktno
pyinstaller radiola.spec --clean --noconfirm

3. Testiranje

cd dist/Radiola
./Radiola

4. AppImage — distribucija na vse Linux sisteme

Namesti appimagetool: 
snap install appimagetool --classic
# ali prenesi direktno:
wget https://github.com/AppImage/AppImageKit/releases/download/continuous/appimagetool-x86_64.AppImage
chmod +x appimagetool-x86_64.AppImage
Zaženi build.py — samodejno ustvari AppImage če appimagetool obstaja: 
python3 build.py
# Rezultat: Radiola-1.4.0-x86_64.AppImage
Testiranje AppImage: 
chmod +x Radiola-1.4.0-x86_64.AppImage
./Radiola-1.4.0-x86_64.AppImage

mpv namestitev (za končne uporabnike)

# Ubuntu/Debian
sudo apt install mpv

# Fedora
sudo dnf install mpv

# Arch
sudo pacman -S mpv

Pogoste težave

NapakaRešitev
_tkinter not foundsudo apt install python3-tk
Manjkajoče knjižniceGradi na najstarejši podprti distribuciji (Ubuntu 20.04 LTS)
AppImage se ne zaženechmod +x *.AppImage in zaženi z ./
FUSE not foundsudo apt install libfuse2 ali ./App.AppImage --appimage-extract-and-run

macOS — .app bundle

1. Odvisnosti

# Homebrew (če ni nameščen)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Python in mpv
brew install python mpv

# Python paketi
pip3 install pyinstaller pillow requests

2. Build

python3 build.py
# Rezultat: dist/Radiola.app

3. Distribucija

# Ustvari DMG (potrebuje create-dmg)
brew install create-dmg
create-dmg \
  --volname "Radiola" \
  --window-size 400 200 \
  --icon "Radiola.app" 100 100 \
  "Radiola-1.4.0.dmg" \
  "dist/"
Gatekeeper: macOS bo blokiral nepotrjene aplikacije. Uporabniki morajo iti v Sistemske nastavitve → Zasebnost in varnost ter dovoliti odpiranje. Za pravo distribucijo je potreben Apple Developer certifikat (~$99/leto).

GitHub Actions — samodejni build za vse platforme

Ko ustvariš GitHub tag (npr. v1.4.0), GitHub samodejno zgradi Radiolo za Windows, Linux in macOS ter objavi Release.

Inicializiraj GitHub repozitorij (če še ni): 
cd ~/radiola
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/TvojeIme/radiola.git
git push -u origin main
Ustvari mapo za workflow: 
mkdir -p .github/workflows
cp build.yml .github/workflows/build.yml
git add .github/
git commit -m "Add GitHub Actions build workflow"
git push
Ustvari Release z novo verzijo: 
# Posodobi VERSION v radiola.py na "1.4.1"
git add radiola.py
git commit -m "Bump version to 1.4.1"
git tag v1.4.1
git push origin main --tags
GitHub Actions samodejno:
  • Zgradi Windows .exe na windows-latest
  • Zgradi Linux binary + AppImage na ubuntu-22.04
  • Zgradi macOS .app na macos-latest
  • Objavi GitHub Release z vsemi datotekami
Preveri build na: github.com/TvojeIme/radiola/actions

Struktura repozitorija

radiola/
├── .github/
│   └── workflows/
│       └── build.yml       ← GitHub Actions workflow
├── radiola.py
├── radiola.spec            ← PyInstaller spec
├── build.py                ← lokalni build skript
├── requirements.txt
├── .gitignore
├── LICENSE
├── README.md
├── CHANGELOG.md
├── ICONS/
│   └── .gitkeep
└── docs/
    ├── radiola_navodila.html
    └── packaging_guide.html

Ročni zagon brez taga

Workflow ima workflow_dispatch — zaženeš ga ročno v GitHub → Actions → Build Radiola → Run workflow.

Brezplačno: GitHub Actions je brezplačen za javne repozitorije. Za zasebne repo-je je na voljo 2000 minut/mesec brezplačno (en build = ~10 minut).
