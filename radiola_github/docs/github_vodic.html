1
Namesti Git na Linux PC

Git je orodje ki sledi spremembam v kodi in jo pošilja na GitHub.

sudo apt install git

# Preveri namestitev
git --version
# Prikaže: git version 2.x.x  ← OK

Nastavi svoje ime in email (GitHub bo to prikazal):

git config --global user.name  "Tvoje Ime"
git config --global user.email "tvoj@email.com"
Enkrat za vedno — te nastavitve se ne ponovijo.

4
Pripravi datoteke in naloži na GitHub

4a — Pripravi strukturo datotek

Ustvari mapo za projekt in vanjo daj vse datoteke:

# Ustvari mapo projekta
mkdir -p ~/radiola/.github/workflows
mkdir -p ~/radiola/ICONS
mkdir -p ~/radiola/docs

Kopiraj datoteke (prilagodi poti kjer imaš shranjene):

# Prilagodi ~/Downloads/Radiola/ na kje imaš datoteke
cp ~/Downloads/Radiola/radiola.py        ~/radiola/
cp ~/Downloads/Radiola/radiola.spec       ~/radiola/
cp ~/Downloads/Radiola/build.py           ~/radiola/
cp ~/Downloads/Radiola/requirements.txt   ~/radiola/
cp ~/Downloads/Radiola/README.md          ~/radiola/
cp ~/Downloads/Radiola/CHANGELOG.md       ~/radiola/
cp ~/Downloads/Radiola/CHANGELOG.md       ~/radiola/

# .gitignore — preimenuj gitignore.txt → .gitignore
cp ~/Downloads/Radiola/gitignore.txt      ~/radiola/.gitignore

# LICENSE — preimenuj LICENSE.txt → LICENSE
cp ~/Downloads/Radiola/LICENSE.txt        ~/radiola/LICENSE

# GitHub Actions workflow
cp ~/Downloads/Radiola/build.yml          ~/radiola/.github/workflows/build.yml

# Navodila
cp ~/Downloads/Radiola/radiola_navodila.html  ~/radiola/docs/
cp ~/Downloads/Radiola/packaging_guide.html   ~/radiola/docs/

Ustvari .gitkeep v ICONS da mapa ostane v Git:

touch ~/radiola/ICONS/.gitkeep

Preverjena struktura

~/radiola/ ├── .github/ │ └── workflows/ │ └── build.yml ← GitHub Actions ├── docs/ │ ├── radiola_navodila.html │ └── packaging_guide.html ├── ICONS/ │ └── .gitkeep ├── radiola.py ← glavni program ├── radiola.spec ← PyInstaller config ├── build.py ← build skript ├── requirements.txt ├── .gitignore ← brez pike v imenu ├── LICENSE ← brez končnice ├── README.md └── CHANGELOG.md

4b — Poveži z GitHub in naloži

# Pojdi v mapo projekta
cd ~/radiola

# Inicializiraj Git
git init

# Poveži z GitHub repozitorijem
# (zamenjaj "TvojeIme" s tvojim GitHub uporabniškim imenom)
git remote add origin https://github.com/TvojeIme/radiola.git

# Dodaj vse datoteke
git add .

# Prikaži kaj bo naloženo (opcijsko preverjanje)
git status

# Ustvari prvi commit (shrani stanje)
git commit -m "Radiola v1.4.0 — začetna objava"

# Nastavi glavno vejo in naloži na GitHub
git branch -M main
git push -u origin main
GitHub te bo vprašal za geslo: Od 2021 GitHub ne sprejema navadnih gesel — potrebuješ Personal Access Token. Glej korak 4c.

4c — Personal Access Token (namesto gesla)

Na GitHub → klikni svojo profilno sliko (zgoraj desno) → Settings
Levo spodaj: Developer settings
Personal access tokens → Tokens (classic) → Generate new token (classic)
Izpolni:
PoljeVrednost
Noteradiola-upload
Expiration90 days ali No expiration
Scopes✅ repo (vse pod repo)
Klikni Generate token → TAKOJ kopiraj token (prikaže se samo enkrat!)
Ko Git vpraša za geslo → prilepi token namesto gesla
Shrani token v varno mesto (npr. KeePass, Bitwarden). Naslednjič ko boš pushiral, ga bo Linux shranil samodejno.

4d — Shrani token da ga ne rabiš vpisovati vsakič

# Linux bo shranil token za prihodnje ukaze
git config --global credential.helper store

# Po naslednjem push → token se shrani in ne bo več vpraša
✓ Koda je na GitHubu! Odpri github.com/TvojeIme/radiola in jo vidiš.
5
Preveri da GitHub Actions deluje
Na GitHub pojdi na tvoj repozitorij → zavihek Actions
Vidiš workflow "Build Radiola" → trenutno ne bo zagnan ker čaka na tag
Preveri da je .github/workflows/build.yml vidna pod Code zavihkom
Workflow se zažene samo ko ustvariš Git tag (npr. v1.4.0). To naredimo v naslednjem koraku.
6
Ustvari Release — GitHub zgradi EXE samodejno

Ko si pripravljen za objavo nove verzije:

# Pojdi v mapo projekta
cd ~/radiola

# Ustvari Git tag z verzijo
git tag v1.4.0

# Pošlji tag na GitHub (to sproži build)
git push origin --tags

Kaj se zgodi na GitHub (samodejno, ~10 minut):

GitHub Actions zazna tag v1.4.0 in zažene 3 vzporedne buildi
Windows strežnik → namesti Python + PyInstaller → zgradi Radiola.exe
Linux strežnik → zgradi binarni + AppImage
macOS strežnik → zgradi .app bundle
Vse 3 datoteke se naložijo v GitHub Release

Preveri napredek:

# Na GitHub → zavihek Actions → klikni "Build Radiola"
# Vidiš 3 vzporedne jobi — zelena kljukica = uspeh

Prenesi EXE:

GitHub → Releases (desno na glavni strani repozitorija)
Klikni v1.4.0
Pod Assets vidiš:
  • 📦 Radiola-1.4.0-Windows.zip — Windows EXE
  • 📦 Radiola-1.4.0-Linux.tar.gz — Linux binary
  • 📦 Radiola-1.4.0-x86_64.AppImage — Linux AppImage
  • 📦 Radiola-1.4.0-macOS.tar.gz — macOS
🎉 Windows EXE je na voljo za prenos — brez da bi imel Windows!

🔄 Za vsako naslednjo verzijo

Ko spreminjaš Radiolo in hočeš objaviti novo verzijo:

cd ~/radiola

# 1. Kopiraj posodobljeno radiola.py
cp ~/Downloads/Radiola/radiola.py .

# 2. Posodobi CHANGELOG.md z novostmi (opcijsko)

# 3. Dodaj spremembe v Git
git add .
git commit -m "v1.5.0 — opis sprememb"

# 4. Pošlji na GitHub
git push

# 5. Ustvari novo verzijo in sproži build
git tag v1.5.0
git push origin --tags

# → čez 10 minut je nov EXE v Releases!

🔧 Pogoste napake

NapakaRešitev
remote: Repository not found Preveri URL: git remote -v — mora biti https://github.com/TvojeIme/radiola.git
Authentication failed Uporabi Personal Access Token namesto gesla (korak 4c)
Updates were rejected Najprej: git pull origin main --rebase nato git push
GitHub Actions ne dela Preveri zavihek Actions → klikni na rdeč X za podrobnosti napake
Build traja preveč Normalno je 5–12 minut. Osveži stran.
tag already exists Izbriši star tag: git tag -d v1.4.0 && git push origin :refs/tags/v1.4.0
